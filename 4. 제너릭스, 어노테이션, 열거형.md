# Generics, Annotation, Enums

## Generics

### Generics란?
Generics란 다양한 타입의 객체드를 다루는 메서드나 컬렉션 클래스에 컴파일시의 타입체크를 해주는 기능이다.

### Generics의 장점
1. 객체의 타입 안정성을 높인다.
2. 형변환의 번거로움을 줄인다.

### Generic 클래스의 선언
클래스 네임 옆에 <제네릭변수>를 넣고, 제네릭 타입이 들어가야 할 위치에 제네릭 변수를 넣어준다.

|변수|설명|
|-|-|
|E|Element|
|K|Key|
|N|Number|
|T|Type|
|V|Value|

+ JDK1.7 부터는 추정이 가능한 경우 타입 생략 가능
+ 제네릭 타입에 들어갈 변수의 타입을 제한하고 싶다면 extends를 사용

### Generic 클래스의 제한점
1. Static 멤버에 타입변수 사용 불가
2. Generic 타입의 배열 생성 X

## 와일드카드

### 와일드카드가 필요한 이유
제네릭 클래스에서 상위/하위타입 알아보기
- 오직 raw-type간에만 상하위 관계가 생기고, 제네릭 타입에서는 상하위 관계가 적용되지 않는다.

### 와일드카드란?
제네릭 타입과 관계없이 메서드의 파라미터로 가져와 사용할 수 있게함.

## Annotation

### 어노테이션이란?
마치 주석과 같이 프로그래밍 언어에 영향을 미치지 않으면서 다른 프로그램에 유용한 정보를 제공하는 것.

### 자바에서 제공하는 표준 어노테이션
